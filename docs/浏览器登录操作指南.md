# 浏览器登录操作指南

## 背景

Agent 使用 Playwright 启动一个专属浏览器（独立 profile），与你平时用的 Chrome 完全隔离。
**首次启动或 profile 丢失后**，需要手动登录一次智联招聘，之后登录态会持久保存，无需再次登录。

---

## 一、正确的启动方式

### 推荐：24 小时守护进程（浏览器一直开着）

在 **Mac 系统终端**（Terminal.app 或 iTerm，不要用 Cursor 内嵌终端）运行：

```bash
cd "/Users/huaiyuan/Library/Mobile Documents/com~apple~CloudDocs/Tan工作资料/Oracle公司/PPT/cursor/openclaw" && npm run agent:daemon
```

启动后：
1. 浏览器自动弹出
2. 终端打印 `[等待登录] 未检测到登录态，请在浏览器中完成登录...`（如果尚未登录）
3. 在弹出的浏览器里完成登录（扫码或账号密码）
4. 终端打印 `[等待登录] 登录成功，继续执行...`
5. Agent 开始自动工作，**每 1 分钟**扫一次互动区，浏览器不会关闭
6. Ctrl+C 停止

## 二、登录态如何保持

- 登录 Cookie 保存在 `./data/browser-profile/` 目录（Agent 专属 profile）
- 只要该目录存在且未被清除，后续启动无需再次登录
- 登录后**不要手动删除** `./data/browser-profile/` 目录

---

## 三、常见问题与解决方法

### 问题 1：启动后浏览器一直在等待登录

**现象：** 终端持续打印 `[等待登录] 已等待 Xs...`

**原因：** Profile 里没有有效的登录 Cookie（首次使用、或 profile 被清除）

**解决：** 在弹出的浏览器里手动登录智联招聘，登录完成后系统自动继续

---

### 问题 2：ProcessSingleton 错误，浏览器启动失败

**现象：** 日志出现 `Failed to create a ProcessSingleton for your profile directory`

**原因：** 同一个 profile 目录被两个浏览器实例同时锁定

**解决：**
```bash
# 杀掉所有残留的 Playwright 浏览器进程
pkill -f "Google Chrome for Testing" 2>/dev/null || true
# 删除锁文件
rm -f "./data/browser-profile/SingletonLock"
rm -f "./data/browser-profile/SingletonCookie"
# 重新启动
npm run agent:daemon
```

---

### 问题 3：状态文件损坏（`Unexpected end of JSON input`）

**现象：** 启动时报错 `Unexpected end of JSON input`，程序立即退出

**原因：** `data/recruit-agent-state.json` 文件为空或内容不完整（进程被强制杀死时可能发生）

**解决：**
```bash
echo '{"candidates":{},"contactCounters":[],"handoverQueue":[],"handledInteractions":[],"consecutiveErrors":0}' > "./data/recruit-agent-state.json"
```

然后重新运行。代码已做容错处理，新版本遇到此情况会自动重置，无需手动操作。

---

### 问题 4：互动区点了但没有回复

**现象：** Agent 点开每个会话，但没有发送任何消息

**可能原因：**
1. **发送延迟**：早期版本 `minDelayMs=45000`，即每次发消息前等 45 秒，看起来像没动作。当前版本已改为 3-8 秒。
2. **重复跳过**：该消息已在上轮处理过，本轮自动跳过（正常行为）
3. **Agent 自发消息**：会话最后一条是 Agent 自己发的，系统识别后跳过（防止自言自语）

**解决：** 确认 `config/recruit-agent.json` 里 `guardrails.minDelayMs` 值为 `3000`，不是 `45000`。

## 四、注意事项

| 事项 | 说明 |
|------|------|
| 不要用 Cursor 内嵌终端启动 | Cursor 内嵌终端有 `Aborted` 环境限制，请用系统 Terminal |
| 不要手动打开同一个 profile | 会产生 ProcessSingleton 锁冲突 |
| Mac 不要进入休眠 | 休眠会暂停进程，电源插着 + 关闭休眠最稳定 |
| 浏览器不需要在前台 | 可锁屏、可切换到其他 App，Agent 照常工作 |
| 网络要保持连接 | 断网会导致页面操作失败 |
