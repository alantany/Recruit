# 主线与命令对照（执行速查）

默认配置文件：`config/recruit-agent.json`。以下命令均在项目根目录执行，且需先完成 `npm install`、`npm run playwright:install` 与浏览器登录（见 `docs/010005-招聘Agent使用手册.md`）。

**Windows：** 日常跑主线 A 可用根目录 **`run.bat`**（`git pull` + 依赖 + `agent:search-interaction-loop`）。**`deploy-windows.bat`** 仅用于首次环境部署（装 Node/Git、装依赖、build、init 等），不必每次执行；区别见 `docs/010005-招聘Agent使用手册.md` §3。

**macOS / Linux（开发机）：** 根目录 **`run.sh`** 不执行 `git pull`，仅 `npm install` + 主线 A，避免本地被远程覆盖（首次 `chmod +x run.sh`，再 `./run.sh`）。

---

## 两条业务主线

| 主线 | 含义 | 一键/推荐命令 | 说明 |
|------|------|----------------|------|
| **A** | 关键字搜索 → 打招呼/要简历 → 互动跟进 | `npm run agent:search-interaction-loop` | **不含**推荐人才、潜在人才、同步职位；循环：搜索 ↔ 互动 |
| **B** | 推荐 / 潜在 → 评分 → 触达 → 互动 | 无单一 npm 名；见下方「分步」或全量 `workflow` | 推荐、潜在分两条子命令；互动与 A 共用同一套 `interaction:run` |

---

## 主线 A：命令对照

| 目的 | npm | 等价（可自行改 `--config`） |
|------|-----|------------------------------|
| 搜索 ↔ 互动循环（主推） | `npm run agent:search-interaction-loop` | `tsx src/recruit-agent/main.ts search-interaction:loop --config config/recruit-agent.json` |
| 仅一轮搜索人才 | `npm run agent:search` | `... search:run ...` |
| 仅一轮互动区 | `npm run agent:interaction` | `... interaction:run ...` |

注意：`search-interaction:loop` 需 `automation.autoWorkEnabled=true` 才会在每轮跑互动；否则只循环搜索。默认 `searchInteractionLoop.maxRounds=0` 表示**无限往返**，用 **Ctrl+C** 结束进程；若改为正整数则跑满该轮数后自动退出。

---

## 主线 B：命令对照

| 目的 | npm | 等价 |
|------|-----|------|
| 推荐人才（评分 + 触达） | `npm run agent:recommend` | `... recommend:run ...` |
| 潜在人才（更高分才触达） | `npm run agent:potential` | `... potential:run ...` |
| 互动区（与主线 A 相同实现） | `npm run agent:interaction` | `... interaction:run ...` |

推荐/潜在当前实现：**只处理第一个激活岗位**（多岗位时需人工切换或后续迭代）。

---

## 全量编排与其它

| 目的 | npm | 说明 |
|------|-----|------|
| 一次跑完：职位同步 → 推荐 → 搜索 → 潜在 → 互动 → 日报 | `npm run agent:workflow` | 不是「第三条主线」，是串联多能力 |
| 后台定时多任务（互动 / 推荐 / 搜索 / 潜在 / 职位同步 / 日报等） | `npm run agent:daemon` | 并行定时，非 A/B 单线故事 |
| 仅同步职位中心 | `npm run agent:jobs` | |
| 仅生成日报与报表文件 | `npm run agent:report` | |
| 初始化目录与状态路径 | `npm run agent:init` | 首次部署 |

---

## 与主线无关的底层命令（供脚本/排查）

可直接调用 `tsx src/recruit-agent/main.ts <command> --config config/recruit-agent.json`，其中 `<command>` 取值：`init` | `jobs:sync` | `recommend:run` | `search:run` | `potential:run` | `interaction:run` | `search-interaction:loop` | `workflow:run` | `daemon:run` | `report:daily`。
