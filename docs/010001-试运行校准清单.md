## 试运行校准清单

### 阶段一：影子模式

目标：只验证抓取和打分，不发送任何消息。

配置要求：

- `dryRun = true`
- `dailyContactLimit = 20`
- `hourlyContactLimit = 8`

执行命令：

```bash
npm run agent:jobs
npm run agent:recommend
npm run agent:search
npm run agent:report
```

检查点：

- 是否能稳定抓到候选人卡片
- 姓名、学历、年限、学校、标签是否抽取正确
- 高分候选人是否真的值得联系
- 低分候选人是否真的该被过滤

通过标准：

- 抽取准确率达到你主观判断的 80% 以上
- 没有明显误杀核心候选人
- 没有把大量无关候选人排到前面

### 阶段二：受监督自动化

目标：系统自动决定“会联系谁”和“互动页如何处理”，但你先不真正放开发送。

配置要求：

- 保持 `dryRun = true`
- 调整 `manualReviewScoreMin` 与 `autoContactScoreMin`

执行命令：

```bash
npm run agent:workflow
```

检查点：

- 日志里“would contact”的候选人是否符合你的判断
- 人工复核区间的人是否足够少
- 是否有明显不应该联系的人进入自动触达区
- 是否有回复被正确转入人工接管

校准建议：

- 自动联系过多：提高 `autoContactScoreMin`
- 自动联系过少：降低 `autoContactScoreMin`
- 高分里混入杂质：增加 `mustHaveKeywords`
- 好人被误杀：减少 `excludeKeywords` 或降低硬过滤强度

### 阶段三：小流量真实发送

目标：在真实账号上验证页面稳定性和平台风控。

配置要求：

- 改成 `dryRun = false`
- 每天先控制在 10 到 20 人

执行命令：

```bash
npm run agent:workflow
```

检查点：

- 页面点击是否稳定
- 是否出现误发、串发、重复发送
- 平台是否提示异常操作
- 候选人回复率是否可接受
- 互动页是否出现错误自动回复

停止条件：

- 连续 2 次误点
- 连续 2 次发错对象
- 出现平台安全提示
- 触达回复率明显低于人工基线

### 阶段四：正式运行

放量前确认：

- 页面选择器已稳定一周
- 候选人状态流转正确
- 日报可正常导出
- 人工接管列表可正常导出
- 话术投诉率低
- 账号没有触发明显风控

### 每周复盘建议

每周看 4 个指标：

1. 自动触达人数
2. 回复人数
3. 简历到达人数
4. 明显误触人数

如果“明显误触人数 / 自动触达人数”大于 10%，优先回到影子模式重新校准。
