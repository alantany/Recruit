# 招聘 Agent 开发进度和计划（持续更新）

## 1. 文档定位

本文件用于记录“当前实现到哪里、下一步做什么、按什么标准验收”。  
原则：需求变更后先更新本文件，再进入开发。

## 2. 当前需求快照（2026-03-27）

- 目标平台：智联招聘（业务目标与当前代码实现一致）。
- 核心目标：通过岗位关键字搜索触达候选人，索要简历，下载简历并沉淀到本地知识库。
- 任务约束：
  - 地域：东北地区
  - 批次触达上限：20 人/任务
  - 对话接续：候选人回复后交给互动区既有流程

## 3. 开发进度总览

### 3.1 已完成（可用）

- 搜索/推荐/潜在人选扫描能力已具备。
- 互动区自动处理能力已具备（索简历、拒绝结束、转人工、致谢）。
- 互动区简历下载能力已具备（含附件/在线简历识别）。
- 简历文件归档与证据入库已具备。
- 会话去重与重复回复保护已具备。
- 基础风控（每日/小时上限、随机延迟、错误阈值）已具备。
- 自动干活总开关已具备（可一键关闭互动区自动交流与简历下载，用于核验）。

### 3.2 部分完成（需补齐）

- “岗位关键字搜索 -> 打招呼 -> 索要简历”前链路已具备组件，但需要按新需求做确定性串联。
- 东北地区限制目前依赖岗位城市/匹配规则，需升级为任务级硬过滤。
- 20 人限制当前主要是日级限制，需补充任务批次级限制。

### 3.3 未完成（本轮重点）

- 智联平台关键动作能力核验（打招呼、发消息、下载简历稳定性）。
- “打招呼后是否可直接发送消息”页面行为实测与策略分支实现。

## 4. 本轮实施计划（最小闭环）

### P0（必须完成）

1. **平台能力核验**  
   登录智联，确认“打招呼”与“发送消息”是否可连续自动执行。  
   输出：能力矩阵（可做/不可做/受限条件）。

2. **前链路动作编排**  
   实现任务流：岗位关键字搜索 -> 候选人筛选（东北）-> 打招呼 ->（可发消息时）索要简历。  
   输出：稳定可回放的动作日志。

3. **任务级上限控制**  
   增加“每次任务最多 20 人”硬阈值。  
   输出：任务结束时统计报告（尝试数/成功数/跳过数）。

4. **互动区接续验证**  
   确认候选人回复后可无缝进入互动区已实现逻辑，不重复发首触达消息。  
   输出：端到端样例日志。

### P1（建议）

1. 东北地区城市映射配置化（黑龙江/吉林/辽宁及主要城市）。
2. 触达失败重试策略（轻量重试，避免风控触发）。

## 5. 风险与应对

- 平台页面结构变更风险：优先通过选择器分层与配置化应对。
- 平台限制消息发送风险：若不能直接发消息，回退为“仅打招呼 + 互动区索简历”。
- 同名候选人误匹配风险：优先使用会话稳定键和候选人稳定 ID 去重。

## 6. 验收清单（本轮）

- [ ] 能按岗位关键字在目标平台检索候选人。
- [ ] 候选人过滤命中东北地区。
- [ ] 单任务触达不超过 20 人。
- [ ] 打招呼动作稳定执行。
- [ ] 若平台允许，索要简历消息可自动发送。
- [ ] 候选人回复后进入互动区链路，无重复首触达。
- [ ] 简历可下载并完成本地归档（文件+证据）。
